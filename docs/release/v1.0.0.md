# moyAI v1.0.0

## 日本語

v1.0.0では、ローカルLLMを使う日常の開発作業を、Desktop上で自然に追跡し、安全境界とprovider failureを明確に判断できる形へ整えました。

### 主な変更

- Desktopのcanonical historyを連続した会話として表示し、表示専用の連番と長文時のページ遷移を廃止しました。完了した作業履歴はturn単位で折りたため、左側の履歴railから過去位置へ移動できます。
- Sub Agentを安定したアイコン、task preview、最終状態を持つ説明付きカードへ集約しました。カードを選ぶと、root会話を離れず右paneでchild sessionのread-only実行履歴を確認できます。
- **承認を求める**、**代理で承認**、**フルアクセス**の3種類のpermission modeを追加しました。代理で承認はtask agentと分離したtool-less AI Guardianを使います。
- Windowsのworkspace-mode processをrestricted token、ACL、suspended spawn、Job Objectで実行するnative sandboxを追加しました。初期化や起動に失敗した場合はunrestrictedで再試行せずfail closedにします。
- LM Studio Responses APIのprovider-side generation failureをtypedに分類し、configured `max_output_tokens`を診断へ含めました。不完全なtool callはlocal commit・実行しません。
- LM Studio/Qwen向けの検証済み出力上限を`8192`へ統一しました。semantic compaction後のcursor-less requestはsummaryをuser-context anchorとして再投影し、保持されたassistant/tool suffixから再開してもprompt-template errorにならないようにしました。
- Desktopの末尾付近ではuser送信だけでなくassistant受信中も末尾へ追従し、過去位置を読んでいる場合はscroll位置を保持します。過去履歴の追加時も表示中anchorを維持します。

### 更新時の注意

- 既存DBはV45までmigrationされます。初回起動前にmoyAIのdata directoryをbackupしてください。
- `auto_review` access modeが正式な選択肢として復元されます。既存の`default`と`full_access`はそのまま保持されます。
- tool-heavy runではreasoningとserialized tool-call argumentsも`max_output_tokens`を消費します。LM Studio/Qwenの同梱profileは`8192`を使用します。
- Windows workspace sandboxは、admission時に存在する有限objectを保護するunelevated defenseです。完全なnamespace isolation、firewall、private desktop、same-user host processからの隔離を意味しません。

### 検証

- Rust full suite、Desktop Web test/build、release buildを実行しました。
- LM Studio/Qwenでtool-call出力上限のA/Bと、compaction後のrequest projectionを確認しました。
- actual Tauri Desktopでcase3の同一session継続、履歴・Sub Agent表示、末尾追従、compacted session再開を確認しました。
- 公開zipを展開し、同梱Desktop binaryのvisible smokeとartifact identityを確認しました。

## English

v1.0.0 makes local-LLM development work easier to follow in the Desktop app while making permission boundaries and provider failures explicit.

### Highlights

- Rendered canonical Desktop history as one continuous conversation, removing display-only step numbers and long-conversation pagination. Completed work is collapsible per turn, with a left history rail for jumping to earlier positions.
- Consolidated each Sub Agent into one descriptive card with a stable icon, task preview, and final state. Selecting a card opens the child session's read-only execution history in the right pane without leaving the root conversation.
- Added three permission modes: **Ask for approval**, **Approve for me**, and **Full access**. Approve for me uses a tool-less AI Guardian isolated from the task agent.
- Added a native Windows workspace process sandbox using a restricted token, ACLs, suspended spawn, and Job Objects. Initialization or launch failure is fail-closed with no unrestricted retry.
- Classified provider-side Responses generation failures as typed diagnostics, including the configured `max_output_tokens`. Incomplete tool calls are never committed or executed locally.
- Standardized the verified LM Studio/Qwen output budget at `8192`. After semantic compaction, cursor-less requests project the summary as a user-context anchor so a retained assistant/tool suffix does not trigger a prompt-template failure.
- Added near-tail follow behavior for both user sends and incoming assistant output, while preserving position when the user is reading older history. Prepended history also keeps its visible anchor.

### Upgrade notes

- Existing databases migrate through V45. Back up the moyAI data directory before first launch.
- The `auto_review` access mode is restored as a first-class option; existing `default` and `full_access` values are preserved.
- Tool-heavy runs consume `max_output_tokens` for reasoning and serialized tool-call arguments as well as normal text. The bundled LM Studio/Qwen profile uses `8192`.
- The Windows workspace sandbox is an unelevated, finite existing-object defense. It is not complete namespace isolation, a firewall, a private desktop, or isolation from same-user host processes.

### Verification

- Ran the full Rust suite, Desktop Web tests/build, and release build.
- Verified tool-call output-budget behavior and post-compaction request projection with LM Studio/Qwen.
- Exercised same-session case3 continuation, history/Sub Agent UI, tail following, and compacted-session resume in an actual Tauri Desktop window.
- Extracted the published zip and verified the bundled Desktop binary with a visible smoke test and artifact identity checks.
